Why I Switched to Astro

August 5, 2025 · 3 min read

For years, I’ve built my static sites using plain HTML, or sometimes used heavier frameworks like React, which always felt overkill for a simple blog. That changed when I discovered Astro.

What is Astro?

Astro is a static site builder that lets you write components using your favorite frameworks (React, Vue, Svelte, etc.) but compiles to static HTML by default. That means faster performance and smaller bundles.

Build faster websites with less JavaScript — that’s Astro in one sentence.

Key Reasons I Switched

  • Zero JS by default: Unless you explicitly add interactivity, pages are pure HTML.
  • File-based routing: Just like Next.js — drop files into `src/pages/`.
  • Markdown + MDX: First-class support for blog posts in Markdown with JSX.
  • Built-in optimizations: Image handling, lazy loading, and partial hydration.

My Experience So Far

Within 2 hours, I migrated my blog layout, imported my old markdown posts, and deployed to GitHub Pages. The performance difference was immediate — lighthouse scores jumped to 100 across the board without tweaks.

Should You Switch?

If you care about performance and love writing content in Markdown — give Astro a try. It's minimal, modern, and made for content-driven sites.

Quick Code Sample

npm create astro@latest
cd my-astro-site
npm install
npm run dev

That’s all you need to start.
